  ---
  Tại sao không dùng GitHub API trực tiếp?

  GitHub REST API có thể tạo repo và upload từng file — nhưng không thể nhận git
   push. Để push code lên GitHub qua API, bạn phải Base64-encode từng file và
  gọi PUT /repos/{owner}/{repo}/contents/{path} cho mỗi file một. Với workspace
  có 24 files thì đó là 24 lần gọi API riêng lẻ, không có commit history, không
  có branch — chỉ là upload file thô. Kết quả trông giống Google Drive hơn là
  git repo.

  Tại sao không dùng MCP?

  Nhìn vào danh sách MCP đang kết nối trong Claude Code của bạn: Canva, Gmail,
  Google Drive, Notion, Vercel. Không có GitHub MCP. GitHub có MCP server riêng
  nhưng chưa được cấu hình trong setup của bạn.

  Tại sao CLI là đúng cho trường hợp này?

  Vì code đã nằm trong git repo (git init + 3 commits rồi). Push git commits lên
   GitHub là một git operation — nó dùng git protocol, không phải HTTP REST. gh 
  CLI làm đúng 2 việc cùng lúc: gọi GitHub API để tạo repo mới, sau đó dùng git
  để push commits lên. Đây là flow tự nhiên nhất.

  Tóm lại: API không push được commits. MCP GitHub không có trong setup. CLI là
  cách đúng với git repo.

  settings — nhưng đó là việc khác. Bây giờ CLI là nhanh nhất.
